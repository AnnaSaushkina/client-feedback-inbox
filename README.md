# Инбокс клиентских правок

Внутренняя веб‑панель для учёта клиентских правок из Telegram‑чатов: единый реестр задач, статусы, приоритеты, канбан‑очередь и история изменений.

Проект делается в рамках проектной практики (менторинг).
Цель — повысить прозрачность работы команды и ускорить обработку правок без потери контекста.

## Tech stack

- React + TypeScript
- Vite
- Ant Design
- Vitest

## К реализации:

### MVP — Реестр правок

- [ ] Создание правки: описание + прикрепление скриншота/файла
- [ ] Список (реестр) правок: сортировка/фильтры, быстрый поиск
- [ ] Статусы правки (например: New → In progress → Review → Done)
- [ ] Приоритеты (Low / Medium / High)

### Workflow — Канбан / очередь

- [ ] Канбан‑доска по стадиям
- [ ] Очередь по исполнителям
- [ ] Подсветка “просроченных/зависших” правок (правило просрочки настраивается)

### Карточка правки + контекст

- [ ] Страница/модалка правки с превью вложений
- [ ] Комментарии (по необходимости)
- [ ] Audit log статусов: кто и когда менял статус/приоритет/исполнителя

### Качество

- [ ] Unit/Component tests на ключевые сценарии (создание, смена статуса, приоритеты, подсветка просрочки)
- [ ] Базовые проверки качества: линт/форматирование (опционально)

## Запуск

- Установка зависимостей:

  ```bash
  pnpm install
  ```

- Запуск в dev режиме:
  ```bash
  pnpm dev
  ```
- Сборка:
  ```bash
  pnpm build
  ```
- Предпросмотр production сборки:
  ```bash
  pnpm preview
  ```
- Запуск тестов:
  ```bash
  pnpm test
  ```

## Примечания:

- Источник правок: на текущем этапе предполагается ручное внесение правки из телеграм сообщения/обсуждения в панель.
- Фокус проекта — удобный workflow и сохранение истории изменений.
